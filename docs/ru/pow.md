# Proof-of-Work

DeepSeek требует решения Proof-of-Work задачи перед каждым запросом к chat completion API и перед регистрацией. Клиент делает это автоматически - вмешательства пользователя не требуется.

---

## Что такое PoW в контексте DeepSeek

Перед отправкой запроса клиент получает от сервера **challenge** - объект с полями:

| Поле         | Описание                      |
|--------------|-------------------------------|
| `salt`       | Строка-соль                   |
| `expire_at`  | Время истечения задачи        |
| `difficulty` | Максимальное число итераций   |
| `challenge`  | Ожидаемый хеш (32 байта, hex) |
| `algorithm`  | Алгоритм (`DeepSeekHashV1`)   |
| `signature`  | Подпись задачи                |

Задача: найти целое число `nonce` от `0` до `difficulty` такое, что:

```
Keccak-256(f"{salt}_{expire_at}_{nonce}") == challenge
```

Найденный `nonce` упаковывается в JSON и кодируется в base64, затем отправляется в заголовке `X-DS-PoW-Response`.

---

## Типичная сложность

Типичное значение `difficulty` - **144 000**.

Это означает, что в худшем случае нужно вычислить 144 000 хешей Keccak-256. На практике нужный `nonce` находится в среднем за половину итераций.

## Производительность

### C++ расширение

Расчёты вынесены в модуль `aiodeepseek/pow/_pow.cpp` - C++17 с применением:

- **AVX2** - SIMD-инструкции Intel/AMD для параллельного вычисления 4 хешей одновременно
- **Многопоточность** - задача делится между всеми ядрами CPU через `std::thread`
- **23-раундовый Keccak** - используется нестандартный вариант с 23 раундами вместо 24
- **Частичный статический хеш** - неизменяемая часть входных данных (без nonce) хешируется один раз и переиспользуется

На современном CPU с 8+ ядрами вычисление занимает **3-15 миллисекунд**. Реализация кастомного `Keccak` на чистом Python занимала ~30 секунд (~10 с `multiprocessing`).

### Сборка расширения

```bash
pip install git+ssh://git@github.com/boykopovar/aiodeepseek.git -U
```

Требования для сборки:
- Компилятор C++17 (GCC 7+, Clang 5+, MSVC 2017+)
- Процессор с поддержкой AVX2 (Intel Haswell 2013+, AMD Ryzen 2017+)
- `pybind11 >= 2.12`

На Windows компилятор должен поддерживать `/arch:AVX2`. На Linux/macOS используются флаги `-mavx2 -march=native`.

## Возможные проблемы

### PoW не решается (`-1`)

Если решение не найдено в пределах `difficulty`, выбрасывается `PowNotSolvedError("PoW not solved within difficulty limit")`.


### Высокая нагрузка на CPU

При каждом `ask` / `ask_stream` запускается параллельный перебор нонсов на всех ядрах. Это короткие импульсы нагрузки (миллисекунды). Если это критично - используйте `timeout` для управления ожиданием.
